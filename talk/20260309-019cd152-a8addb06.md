**结论**

- 当前 `Xiangshan/kmhv3` 的 bare-metal 启动路径里，**没有已经接通、可直接复用的 DTB 自动生成链**。  
  入口是 [`build_xiangshan_system()`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/xiangshan.py#L261) 调到 [`makeBareMetalXiangshanSystem()`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/FSConfig.py#L660)，最终 workload 固定是 [`RiscvBareMetal`](/nfs/home/yanyue/workspace/GEM5_review/src/arch/riscv/RiscvFsWorkload.py#L35)，不是 `RiscvLinux`。

- **现成可复用的 DTB 逻辑只存在于通用 RISC-V FS Linux 路径**，核心是：
  - [`generateDtb(system)`](/nfs/home/yanyue/workspace/GEM5_review/configs/example/riscv/fs_linux.py#L100)
  - [`FsLinux::initState()`](/nfs/home/yanyue/workspace/GEM5_review/src/arch/riscv/linux/fs_workload.cc#L45)
  - [`HiFive.generateDeviceTree()`](/nfs/home/yanyue/workspace/GEM5_review/src/dev/riscv/HiFive.py#L201) / [`HiFive.annotateCpuDeviceNode()`](/nfs/home/yanyue/workspace/GEM5_review/src/dev/riscv/HiFive.py#L220)
  - 设备节点实现：[`Clint.generateDeviceTree()`](/nfs/home/yanyue/workspace/GEM5_review/src/dev/riscv/Clint.py#L57)、[`Plic.generateDeviceTree()`](/nfs/home/yanyue/workspace/GEM5_review/src/dev/riscv/Plic.py#L57)、[`RiscvMmioVirtIO.generateDeviceTree()`](/nfs/home/yanyue/workspace/GEM5_review/src/dev/riscv/RiscvVirtIOMMIO.py#L51)、[`RiscvUart8250.generateDeviceTree()`](/nfs/home/yanyue/workspace/GEM5_review/src/dev/serial/Uart.py#L71)

- `kmhv3` 当前 system 里挂的是 [`UartLite`/`Clint`/`NemuMMC`/`NemuPlic`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/FSConfig.py#L678)，其中：
  - `Clint` 有 DT 生成。
  - `UartLite`、`NemuMMC`、`NemuPlic` **没有** `generateDeviceTree()`。
  - 也**没有** `system.platform`；而 CPU DT 节点会依赖 [`BaseCPU.generateDeviceTree()`](/nfs/home/yanyue/workspace/GEM5_review/src/cpu/BaseCPU.py#L248) 去找 `Platform.annotateCpuDeviceNode()`，所以当前 kmhv3 路径连 CPU 节点也拿不到完整注释。

**最小可复用方案**

- 如果目标只是“给 `raw linux.bin` 传 DTB”，**最自然的挂载点是 workload 层，不是 platform 层**：
  - 参数定义放在 [`src/arch/riscv/RiscvFsWorkload.py`](/nfs/home/yanyue/workspace/GEM5_review/src/arch/riscv/RiscvFsWorkload.py#L35) 的 `RiscvBareMetal`
  - 加载逻辑放在 [`src/arch/riscv/bare_metal/fs_workload.cc`](/nfs/home/yanyue/workspace/GEM5_review/src/arch/riscv/bare_metal/fs_workload.cc#L72) 的 `BareMetal::initState()`

- 具体复用方式：**照抄 `FsLinux::initState()` 的 DTB 装载语义到 `BareMetal::initState()`**
  - 读取 `dtb_filename`
  - 可选追加 bootargs
  - 写入 `dtb_addr`
  - 对每个 hart 设 `A1 = dtb_addr`  
  参考现成实现：[`FsLinux::initState()`](/nfs/home/yanyue/workspace/GEM5_review/src/arch/riscv/linux/fs_workload.cc#L45)

- 配置侧最自然的接线点是在 [`configs/common/xiangshan.py`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/xiangshan.py#L261) 的 `build_xiangshan_system()` 返回后、`Simulation.run_vanilla(...)` 前，给 `test_sys.workload` 填 `dtb_filename/dtb_addr`。
