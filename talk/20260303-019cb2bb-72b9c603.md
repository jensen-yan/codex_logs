# SC Test Patterns (G / IMLI)

## Goal
Create micro-tests where TAGE is weak but a specific SC table can fix direction.

## GTable (global history) pattern

Use when branch outcome depends on recent cross-branch outcomes rather than pure local period.

- Keep one target branch PC stable.
- Add 2-4 feeder branches before target branch.
- Make target direction depend on feeder outcomes from previous iterations.
- Inject low-amplitude noise branches to stress TAGE confidence.

Example idea:
- `b0` period 3, `b1` period 5, `b2` period 7.
- Target: `t = last_b0 ^ last_b1 ^ (b2_now & 1)`.
- Expectation: `g_only` can improve some PCs; `l_only` may not.

## ITable (IMLI) pattern

Use when direction depends on loop iteration phase (especially backward-taken count).

- Use fixed loop trip count (e.g., 16/24/32).
- Keep one branch at same static PC inside loop.
- Make one phase-specific branch flip near loop tail/head.
- Optionally alternate outer-phase to shift the flip position.

Example idea:
- For each outer loop:
  - inner `i in [0, 31]`.
  - target taken iff `i == 17` (or `i == 17/18` by phase).

1. `summary.csv`: target profile has `condMiss_delta < 0`.

3. Same hot branch has positive `net_use`.
4. Focus table metrics are meaningful on that branch:
   - `focus_decisive_ratio` not near zero.
   - `focus_agree_fix_ratio` reasonably high.

- Branch fully local periodic -> LTable dominates, G/I hard to show.
- Too much randomness -> all tables degrade, no stable gain.
- Target branch not hot enough -> statistics too noisy.
