**2) 这些文件能支持到什么粒度**

**`topMispredictsByBranch.csv`**
能做到：

- `per-branch` 级别的 PC 排名分析
- 跨 run 比较同一 PC 的 `mispredicts / total / misPermil / dirMiss`

它的表头是：

```csv
pc,type,mispredicts,total,misPermil,dirMiss,tgtMiss,noPredMiss
```

所以它适合做：

- 哪些分支 PC 在 run364 到 run368 之间改善/恶化
- 改善主要是方向错(`dirMiss`)还是 target/noPred

但它**不能**告诉你：

- 这个 branch 的改善来自 SC 的哪个子表
- 某次预测是哪个 slice 决定的

**`stats.txt`**
能做到：

- `per-benchmark` 级别的整体 SC 效果分析
- `per-subtable` 的全局统计归因

其中明确存在这些 `mgsc` 计数器：

- `scUsed`
- `scCorrectTageWrong`
- `scWrongTageCorrect`
- `bwWeightScaleDiff`
- `lWeightScaleDiff`
- `iWeightScaleDiff`
- `gWeightScaleDiff`
- `pWeightScaleDiff`
- `biasWeightScaleDiff`
- `bw/l/i/g/p/bias PercsumCorrect/Wrong`

这说明 `stats.txt` 可以回答：

- SC 是否在这个 benchmark 上真的介入了
- SC 相对 TAGE 是净修复还是净伤害
- 哪个子表在全局上更“decisive”

但它仍然**只是全局聚合计数**，不能下钻到：

- 某个 branch PC
- 某次具体预测
- 某个 slice 对某个 branch 的贡献
