## 1) `-P/--param` 在 kmhv3 的生效链路（Python 配置层）

- `-P/--param` 在通用选项里定义，语义是“相对 `root` 子节点的 Python 赋值表达式”  
  参考：`configs/common/Options.py:193`
- kmhv3 入口会调用 `Options.addCommonOptions(...)`，所以支持 `-P`  
  参考：`configs/common/xiangshan.py:705`
- 运行前，`root.apply_config(options.param)` 会执行这些表达式（`exec`），因此可覆盖脚本里先前赋值  
  参考：`configs/common/Simulation.py:823`、`src/python/m5/SimObject.py:1226`
- `-P` 支持数组切片/多核选择（如 `cpu[0,1,3:8:2]`）  
  参考：`configs/common/Options.py:197`、`src/python/m5/SimObject.py:553`

## 3) 最小可运行命令模板（kmhv3 + raw cpt）

```bash

  --raw-cpt \

  -P "system.cpu[0].branchPred.mgsc.forceUseSC=False" \
  -P "system.cpu[0].branchPred.mgsc.enableLTable=True" \
  -P "system.cpu[0].branchPred.mgsc.enableGTable=False" \
```

## 4) 风险点（实操常见坑）

- `-P` 是 Python `exec`，布尔值必须写 `True/False`（不是 `true/false`）。
- 路径必须从 `root` 子节点开始（这里是 `system...`，不能写 `root.system...`）。
- kmhv3 强制 `bp_type=DecoupledBPUWithBTB`，否则会 `fatal`。  
  参考：`configs/example/kmhv3.py:185`、`configs/common/xiangshan.py:364`
- kmhv3 默认开启 difftest（非 trace 模式），需可用 `ref_so`（如 `GCBV_REF_SO`）。  
  参考：`configs/common/xiangshan.py:720`、`configs/common/xiangshan.py:145`
