**1. warmup/run 指令数现在是哪里定义的**

核心定义在 [`Options.py`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/Options.py#L316)：

- [`--maxinsts`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/Options.py#L316) 默认是 `40*10**6`
- [`--warmup-insts-no-switch`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/Options.py#L337) 默认是 `20*10**6`

真正把这两个值灌到 CPU 上的是 [`Simulation.run_vanilla()`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/Simulation.py#L774)：

- [`testsys.cpu[i].max_insts_any_thread = options.maxinsts`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/Simulation.py#L784)
- [`testsys.cpu[i].warmupInstCount = options.warmup_insts_no_switch`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/Simulation.py#L819)

所以你现在理解的“20M warmup + 20M run”，在当前代码路径下本质上是：

- 总共从 checkpoint 开始执行 40M 指令
- 20M 时做一次 warmup reset
- 剩下 20M 作为统计窗口

这和 [`kmh_v3_btb.sh`](/nfs/home/yanyue/workspace/GEM5_review/util/xs_scripts/kmh_v3_btb.sh#L10) 一致，因为它没有传任何 warmup/maxinst 参数，只是直接调用 `kmhv3.py`。

**2. 能不能只跑前 5M 指令或更短窗口**

可以，而且从代码看是最直接的做法。

原因：

- [`xiangshan_system_init()`](/nfs/home/yanyue/workspace/GEM5_review/configs/common/xiangshan.py#L701) 会把 common options 都加进来
- 所以 `kmhv3.py` 支持直接传 `--maxinsts` 和 `--warmup-insts-no-switch`

也就是说，你可以直接把窗口缩成：

- `--maxinsts=5000000`
- `--warmup-insts-no-switch=0` 或更小值

这会比“默认 20M+20M”快很多。
